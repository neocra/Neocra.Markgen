Vehicles

The vehicle belongs to a User and enables her/him to :

  • retrieves the POI (Point of Interest i.e fueling station, charging point...) according to the vehicle type
  • trigger the driver journey.

The Vehicles Micro-service supports all the Endpoints needed to complete the CRUD (Create, Read, Update, Delete) operations and manage the Vehicle lifecycle.

Create Vehicle

This reference describes how to create a new vehicle through the REST API. At a minimum, you must specify the required properties for the vehicle. You can optionally specify any other writable properties.

Prerequisites

A vehicle is linked to a User by a One-to-many relationship (1 user can have many vehicles). If you want to create a new vehicle, you must :

  • create a User first
  • and provide the userId contained in the callout of the above-mentionned endpoint.

Required parameters

userId : the User identifier in Be:Mo's database.

fullname : The vehicle name, which is often defined by the User Limitation : Be:Mo doesn't realize unicity checks on vehicle fullname entries.

vehicleType : The vehicle type depends on the propulsion system. It can be :

  • Thermic for Internal Combustion Engines (ICE),
  • Electric for Plug-in Eletric Vehicles (PEV)
  • PlugInHybrid for plug-in hybrid electric vehicles (PHEV).

Limitation : Mild hybrid and Full hybrid vehicles (that don't feature plug-in recharging facility) are considered to be Thermic in Be:Mo database model. .

Optional parameters

brand : The vehicle brand or make. Example : "Renault"

model : The vehicle model (depends on the brand). Example : "Zoe"

version : Version of the vehicle model (depends on the brand & model). Example : "R110"

year : The vehicle's model year Example : "2019"

Be:Mo provides an EV referral database available through the endpoint Get vehicles profiles by brand. It enables you to implement drop-down lists in order to avoid users typo and ensure vehicles data consistency.

productCodes : Be:Mo uses codes to distinguish the different products types from each other. It can refer to :

  • for ICE and PHEV, the fuel types. Example : E85 for Ethanol, B7 for Diesel etc.
  • for PEV and PHEV, the attributes of the energy supply (range of power delivered, speed of charge). Example : E_Standard for a Slow charging (> 5h), E_ultraFast for an ultra fast charging (< 30 min) etc.

The complete list of productCodes can be retrieved through the endpoint Get Products List

connectorCodes : Be:Mo uses codes to distinguish the different charging connectors (plug and socket) from each other. The complete list of connectorCodes can be retrieved through the endpoint Get Connectors List Limitation : This field only makes sense for Electric (PEV) and Plug-in hybrid vehicles (PHEV) .

powerMax : The maximum power charge of the vehicle batteries in kWh. Limitation : This field only makes sense for Electric (PEV) and Plug-in hybrid vehicles (PHEV) .
