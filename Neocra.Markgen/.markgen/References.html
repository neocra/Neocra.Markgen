References

The References Microservice consolidates all the endpoints that enables to retrieve information from Be:Mo referral database. It allows to :

  • find the right spelling of internal references codes (related to connectors, products or distribution networks)
  • retrieve all the Points Of Interest (fuel stations, charging pools, parking slots...)
  • access the Electrical Vehicle referral database.

Get products list

Required Parameters

(none)

Output

serviceType: Depending on the context, category value can refer to :

  • charging
  • fueling
  • parking
  • washing

productCode : Be:Mo uses codes to distinguish the different products types from each other. It can refer to :

  • for ICE and PHEV, the fuel types. Examples : E85 for Ethanol, B7 for Diesel etc.
  • for PEV and PHEV, the attributes of the energy supply (range of power delivered, speed of charge). Examples : E_Standard for a Slow charging (> 5h), E_ultraFast for an Ultra fast charging (< 30 min) etc.

name : More User-friendly label of a product code. Examples : Standard for E_Standard, Ultra fast for E_ultraFast

family : subcategory of productCode (depends on the category value). Examples :

  • for category = charging, family value allowed = {Charging}
  • for category = fueling, family values allowed = {Gasoline, Diesel, Other}

Get connectors list

Be:Mo uses codes to distinguish the different charging connectors (plug and socket) from each other. This Endpoint enables to retrieve the list of Electric Vehicle (EV) connectors codes.

Limitation : The connector notion only makes sense for Electric (PEV) and Plug-in hybrid vehicles (PHEV) .

Required Parameters

(none)

Output

code Examples : "iec_62196_t2_combo_cable", "iec_60309_2_three_16_socket".

Get vehicles profiles by brand

Be:Mo provides an EV referral database available. It enables you to :

  • implement drop-down lists (when creating a vehicle) in order to avoid users typo and ensure vehicles data consistency.
  • retrieves the vehicle charging profile in order to better plan a long driving trip (coming soon)

Required Parameters

(none)

Query Parameters

brand : The vehicle brand or make. Example : "Renault"

model : The vehicle model (depends on the brand). Example : "Zoe"

version : Version of the vehicle model (depends on the brand & model). Example : "R110"

limit : the maximum number of results per page

offset : the offset page

Use cases

  • No brand, model nor version entered as query parameter The Endpoint returns the entire list of EV registered in Be:Mo database and their details according to the limit & offset values.

  • One or several field(s) between brand, model or version entered The Endpoint retrieves all the vehicles matching with the query parameter(s) entered in Be:Mo database with their details according to the limit & offset values.

Get places Networks

Retrieves the list of POI's distribution networks.

Required Parameters

(none)

Output

placeNetworkId : a 3-characters string related to the network/brand of POI in Be:Mo's referral database.

placeNetwork: network/brand of POI Examples : Aral , BP, Eni, Total or Shell for fuel service stations.

Get places

Returns the list of all the POIs connected with Be:Mo according to the search criterias (optional query parameters).

optional parameters

serviceType : Depending on the context, category value can refer to :

  • charging for charging pools
  • fueling for fuel stations
  • parking for parking slots (coming soon)
  • washing for car wash stations (coming soon)

countryCode : Country code ISO 3166-1 alpha-2. Example : FR for 'France', US for 'United States of America'

from : Last update date.
