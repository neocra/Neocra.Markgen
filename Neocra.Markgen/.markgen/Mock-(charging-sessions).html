In real life condition, the RFID session proceeds as follow :

  1. It starts with the user presenting his RFID card to a reader in order to authenticate himself & select a charging mode.
  2. The user can plug the car and start the session
  3. The session is running.
  4. Depending on the Charging Point Owner (CPO) infrastructure, real-time information events can be pushed periodically.
  5. The charging session is completed (by the user or the CPO in case of fatal error)
  6. The user can unplug the car
  7. The user receives his charge details records.

To mimick those conditions in Test environments only, Be:Mo provides the Mock microservice. Thanks to this artefact, a Dev Team can mock a dozen of use cases (OK and KO ) which cover a representative sample of real situations. The Team will be able to anticipate :

  • all the session status that its system could face
  • the inability for several CPO to send all or part of update notifications
  • the failure cases etc.

How to mock RFID Sessions ?

Pre-requisites

Prerequisite 1: The offer which enables you to bill the RFID order already exists and stills valid (current date < offer's expiration date).

Prior checks :

  • Get all offers to retrieve all the offers existing with serviceType = "rfidCard" and check their expiration dates (validUntil)
  • Get the price for card ( offerId as parameter) to confirm that your offer returns the correct price.
More details on how to meet the prerequisite 1
  • If your desired offer with serviceType = "rfidCard" doesn't exist
  • OR a similar offer exists BUT it is expired ),
  1. Create a pricing Model which models the mathematical formula you want to run to calculate the card price (or Get all pricingModels to reuse an existing one).
  2. Create an offer (by setting up a later expiration date on validUntil, the previous pricingModelId & serviceType="rfidCard" as parameters)
  3. Save the offerId from previous response payload.

Prerequisite 2: the User to whom you want to issue the RFID card already exists & his status is "active"

Prior check : User status by requesting Get information on a user in case of any doubt.

More details on how to meet the prerequisite 2
  • If the user does not exist, please create his profile first by using Create a User. His status will be "active" (isActive = 'True') by default
  • If the user exists but has been previously disabled (isActive = 'False'), you must reactivate him first, via Enable a User.

Prerequisite 3: The RFID card of the User is activated.

Prior check : You can retrieve the list of the card(s) assigned to the User and its (their) status by requesting get all cards of a User in case of any doubt.

More details on how to meet the prerequisite 3

  1. If the User does not have any card, you must Create a card order for a user . PS : On Testing environment, you can mock a successful order by setting the right parameters in the payload of Create a card order for a user. (cf. 'Mocking card orders' documentation)

  2. In case of success (order status = 'shipped'), a new card has been created with status 'notActivated', you must use the endpoint Activate a card to update status from 'notActivated' to 'activated'.

  • If a card exists but its status is 'notActivated', you must use the endpoint Activate a card to update status from 'notActivated' to 'activated'.
  • If a card exists but its status is 'disabled', you must use the endpoint Enable a card to update status from 'disabled' to 'activated'.

Prerequisite 4 : you know the 'serviceId' or the 'serviceLabel' of the behavior you want to mimick with your mocked RFID-initiated charging session.

More details on how to meet the prerequisite 4 (Referral table of mocked charging scenarios)

How to retrieve mocked RFID sessions ?

All the sessions can be retrieved thanks to the endpoints of Sessions Microservice. The Rfid-initiated (real or mocked) sessions are identified by the tag isRfid = true

Operation HTTP Method Endpoint name Availability
Read GET Get all sessions for a given user All environments
Read GET Get session information All environments

Important Outputs

isRfid : indicates if the session has been or not initiated by a RFID card.

  • 'False' for a digital session (initiated by the endpoint Start a session).
  • 'True' for a RFID-triggered session (or a mocked RFID session in Testing environments cf. Start mocked charging session with given card)
