Sessions

Depending on the context, a session (or service session) represents :

  • the execution of a charge service
  • the filling of a fuel tank
  • the reservation of a parking spot
  • the use of a car wash station etc.
More details on charging session

Charging session

Before the charge service, the driver must find a charging point or Electric Vehicle Supply Equipment (EVSE), which :

  • delivers the right energy supply (in terms of voltage, AC/DC, power max)
  • provides an appropriate connector to plug his/her car.

The lifecycle of a charging session can be described below :

  1. it starts with the user authentication thanks to a RFID-enabled badge reader or by using an App.
  1. The user can plug the car and start the session
  1. The session is running. Depending on the Charging Point Owner (CPO) infrastructure, the user can receive real time information about the active session.
  1. Stop the charging session (instigated by the user or the Charging Point Owner)
  1. The user can unplug the car
  1. The user receives his charge details records.

The Sessions Micro-service is the second part of the driver journey. It supports all the Endpoints needed to :

  • start a session
  • end a session
  • retrieve sessions history
  • retrieve a given session (active or ended)
Operation HTTP Method Endpoint name Availability
Start POST Start a session All environments
Stop POST Stop a session All environments
Read GET Get all sessions for a given user All environments
Read GET Get session details All environments

Prerequisites :

  1. Create a User
  2. Create a vehicle
  3. Get places around a given position
  4. Get service information

Start a session

This endpoint enables to start digital sessions.

Required parameters

userId : user's identifier in Be:Mo's data base. contained in the callout of the endpoint create a User (Prerequisite #1)

vehicleId : vehicle's identifier in Be:Mo's data base. contained in the callout of the endpoint Create a vehicle (Prerequisite #2)

longitude : longitude coordinate of the Point Of Interest (POI) location. Output from Get places around a given position (Prerequisite #3)

latitude : latitude coordinate of the Point Of Interest (POI) location. Output from Get places around a given position (Prerequisite #3)

serviceId : service's identifier in Be:Mo's data base. Output from Get service information (Prerequisite #4)

Important Outputs

sessionId : The session identifier in Be:Mo's database

status : The session status.

isRfid : indicates if the session has been or not initiated by a RFID card.

  • 'False' for a digital session (initiated by the endpoint Start a session).
  • 'True' for a RFID-triggered session (or a mocked RFID session in Testing environments cf. Start mocked charging session with given card)

Stop a session

This endpoint enables to stop a previously started digital session.

Prerequisites

  1. Start a session Limitation : Session status is new, pending or active.

Required parameters

userId : user's identifier in Be:Mo's data base. contained in the callout of the endpoint create a User (Prerequisite #1)

sessionId : The session identifier in Be:Mo's database. Output from Start a session (Prerequisite #5)

Get sessions history for a given user

Retrieves a list of all the sessions initiated by a given user on a given period.

Prerequisites

The user has initiated at least 1 session.

Required parameter

userId : user's identifier in Be:Mo's data base.

Optional parameter

active : True for an active session.

Important Outputs

isRfid : indicates if the session has been or not initiated by a RFID card.

  • 'False' for a digital session (initiated by the endpoint Start a session).
  • 'True' for a RFID-triggered session (or a mocked RFID session in Testing environments cf. Start mocked charging session with given card)

Other session details (when available)

Get session details

Retrieves the detailed information of a given session

Required parameters

userId : user's identifier in Be:Mo's data base.

sessionId : The session identifier in Be:Mo's database.

Optional parameter

active : True for an active session.

Important Output

isRfid : indicates if the session has been or not initiated by a RFID card.

  • 'False' for a digital session (initiated by the endpoint Start a session).
  • 'True' for a RFID-triggered session (or a mocked RFID session in Testing environments cf. Start mocked charging session with given card)
