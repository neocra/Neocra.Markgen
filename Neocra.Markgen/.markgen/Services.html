Services

The Be:Mo operational terminology distinguishes the notions of product and service from each other :

  • Product refers to the "articles" from the catalog of a fuel station, charging pool, car wash station, parking lot. Example : Ethanol (E85), Gasoline (S95) for fueling, Standard charging, Ultrafast charging...
  • Service refers to the product delivery. Example : execution of a charge service, filling of a fuel tank

The Services Micro-service is the fist part of the driver journey. It supports all the Endpoints needed to :

  • retrieve the nearest Points Of Interest (POI) related to a given service (fuel stations, charging pools, parking slots, car washers) around a location
  • retrieve service information (codes, availability, prices).

Get clustered places around a given position

Retrieves the nearest POI around a given location, grouped into clusters, according to given criterias (search radius, desired product, distribution network...). Depending on the zoom level (from 1 to 12), this endpoint enables to visualize on a map a reduced number of clusters instead of tens, hundreds or thousands single points.

Prerequisites

  1. Create a User
  2. Create a vehicle (optional)
  3. Get product list
  4. Get connector list (for Electric & Plug-in Hybrid vehicles)

Required parameters

userId : user's identifier in Be:Mo's data base. contained in the callout of the endpoint create User (Prerequisite #1)

Optional parameters

zoom : zoom level value (from 1 to 12). If no value is entered, zoom = 1 (the wider zoom range) is set by default.

vehicleId : vehicle's identifier in Be:Mo's data base. contained in the callout of the endpoint Create vehicle (Prerequisite #2)

Longitude : longitude coordinate of User location (current position of the User device or entered manually)

Latitude : latitude coordinate of User location (current position of the User device or entered manually)

productCodes : Be:Mo uses codes to distinguish the different products types from each other. It can refer to :

  • for ICE and PHEV, the fuel types. Example : E85 for Ethanol, B7 for Diesel etc.
  • for PEV and PHEV, the attributes of the energy supply (range of power delivered, speed of charge). Example : E_Standard for a Slow charging (> 5h), E_ultraFast for an Ultra fast charging (< 30 min) etc. The complete list of productCodes can be retrieved through the endpoint Get Products List (Prerequisite #3)

connectorCodes : Be:Mo uses codes to distinguish the different charging connectors (plug and socket) from each other. The complete list of connectorCodes can be retrieved through the endpoint Get Connectors List (Prerequisite #4) Limitation : This field only makes sense for Electric (PEV) and Plug-in hybrid vehicles (PHEV) .

Distance : search radius in kilometers (km)

PlaceNetworkCode : network/brand of POI Example : Aral , BP, Eni or Shell for fuel service stations.

AvailableProduct : True to filter the results and return only available products per place.

DigitalJourney : True to filter the results and return the places enabling digital journey (through a third-party application).

Important output

mode : Cluster indicates if the payload is related to a single places visualization (mode = place) or a clustered one (mode = cluster)

productId : identifier of the product in the POI's database.

count : number of places/stations grouped into the cluster.

attributes : list of capabilities & connectors provided by the station.

Attribute label Type Meaning
capability.charging_profile_capable Capability The CPO is able to provide real time updates of the charging session and charging profile
capability.credit_card_payable Capability Charging at this EVSE can be payed with credit card.
capability.remote_start_stop_capable Capability The station is equipped with terminals allowing digital charging (the session can be triggered remotely by a Mobile/In-car app )
capability.reservable Capability The CPO enables its users to reserve a charging point X minutes before their arrival at the station.
capability.rfid_reader Capability The station is equipped with RFID card reader devices
capability.unlock_capable Capability Connectors have mechanical lock that can be requested to be unlocked.
connector.XXXXXXXX Connector The station is equipped with the mentioned type of connector. Example : connector.domestic_e_socket if at least, one of the charging terminal is equipped with the domestic E socket

Get service information

Retrieves the service details (Id, availability, charging terminals details, tariffs...) related to a productId.

Prerequisite

  1. Get places around a given position or Get clustered places around a given position

Required parameters

userId : user's identifier in Be:Mo's data base. contained in the callout of the endpoint create User (Prerequisite #1)

vehicleId : vehicle's identifier in Be:Mo's data base. contained in the callout of the endpoint Create vehicle (Prerequisite #2)

productId : identifier of the product in the POI's database. contained in the callout of the endpoint Get clustered places around a given position or Get places around a given position (Prerequisite #5).

Important outputs

physicalEquipments : list of connectors available in the station

connectors : for each physical equipment, this output lists the following attributes :

  • standard : standard name (examples : CHAdeMO, Domestic E...) cf. 'References' documentation > section 'Get connectors list'
  • format : connector type (cable or socket)
  • powerType : AC or DC
  • voltage : voltage expressed in volts (V)
  • amperage : amperage expressed in ampers (A)
  • power : power expressed in kilowatts (kW)
  • tariffId : identifier of product tariff in tariff referral grid. Reference Endpoint coming soon.
